# チャットボット ナレッジ更新・改善ガイドライン（AIエージェント向け指示書）

本指示書は、SPLYZAの各種チャットボット（SPLYZA Motion 等）のヘルプナレッジを更新・最適化するための標準ワークフローです。
担当するAIエージェントは、必ず以下のステップ1〜7の順序に従って作業を実行してください。なお、更新元となる最新のヘルプページデータ（PDFファイル）はすでにローカルに準備されています。

## Step 1: リポジトリの最新化と環境確認
1. 対象リポジトリのディレクトリに移動し、`git status` および `git pull` を実行して最新のコードを取得してください。
2. `src/data/knowledge/` ディレクトリ（またはそれに準ずるナレッジ格納場所）にどのようなMarkdownファイルが存在しているか一覧を確認してください。

## Step 2: 最新PDFの読み込みと差分チェック
1. ユーザーから提供される「最新ヘルプページのPDFファイル群」を `Read` ツールで読み込んでください。
2. 現在のナレッジ（Markdownファイル）の内容とPDFを突き合わせ、以下の点を確認してください。
   - 不足しているコンテンツ（新機能や新規ページ）
   - 古くなっているコンテンツ（廃止された手順など）
   - 日本語表現が不自然な箇所

## Step 3: コンテンツの追加・修正と「自然な日本語」への推敲
1. PDFの内容を元に、ナレッジのテキストを最新化してください。
2. 元のPDF（ヘルプページ）の日本語表現に違和感がある場合（例：「自分だけ動画は配信されず見ることができません」等）、チャットボットの回答として自然で分かりやすい日本語（例：「自分だけ動画が見られません（配信されません）」）に推敲してMarkdownを記述してください。
3. ユーザーが次に取るべきアクション（サポートへの連絡や設定変更）を明確に記載してください。

## Step 4: ファイル構成と「見出しの並び順」の完全一致化（最重要）
AIの回答精度には影響しませんが、**人間の今後のメンテナンス性を担保するために極めて重要なステップ**です。
1. 各Markdownファイル内の見出し（`##` や `###`）の並び順を、Notionサイト（PDFの目次）と**完全に一致**させてください。
2. 誤ったカテゴリ（別ファイル）に混ざっている項目があれば、正しいファイルへ移動（引っ越し）させてください。

## Step 5: 「例外事項」の定義ファイルの作成
1. ナレッジディレクトリ内に `EXCLUDED_CONTENTS.md` （除外リスト）を作成してください。
2. PDF上に目次としては存在するが、**「YouTube動画が貼られているだけでテキスト解説がないページ」**を洗い出し、「テキスト解説がないため意図的に除外している」旨をこのファイルに記録してください。
   - ※ これにより、今後の更新担当者やAIが「ファイルが欠落している」と誤認するのを防ぎます。
3. その他、AIの回答精度を上げるためにあえて特殊な見出し構成にしているものがあれば、同様に記録してください。

## Step 6: 回答精度向上（ハルシネーション対策）のためのAPI改修
ナレッジをAIに読み込ませる際の結合処理を改修し、AIがファイル間の境界線を正確に認識できるようにします。
1. `src/app/api/query/route.ts` （または該当するAPIファイル）を開いてください。
2. 複数のMarkdownファイルを1つのテキストに結合している処理を探してください。
3. 単なるハイフン区切り（`--- FILE: filename ---`）になっている場合、以下の**XMLタグ形式**に書き換えてください。
   ```typescript
   // 変更前
   knowledgeBase += `\n--- FILE: ${file} ---\n${content}\n`;
   
   // 変更後（LLMが境界を正確に認識できる形式）
   knowledgeBase += `\n<document title="${file}">\n${content}\n</document>\n`;
   ```

## Step 7: 最終確認とデプロイ（Vercel）
1. `git diff` で意図しない変更が含まれていないか最終確認を行ってください。
2. 変更内容を要約したコミットメッセージを作成し、`git add .` -> `git commit -m "..."` -> `git push` を実行してください。
3. Vercelの自動デプロイが走るため、プッシュ完了をもって本作業は終了となります。

---
**※ 注意事項 ※**
1. **チャットボットの回答への影響確認（必須）**: 
   ナレッジの構成変更やシステムプロンプトの改修など、実際のチャットボットの「回答の質・精度・内容」にネガティブな影響（悪影響）を与えそうな要因を発見した場合や、判断に迷う変更を行う場合は、**必ず事前にその旨をユーザー（担当者）に報告し、確認・了承を得てから作業を進めてください**。独断での危険な変更は厳禁です。
2. **システムプロンプトの制約遵守**: 
   システムプロンプトに「Markdown（太字や見出し）の出力禁止」などの特殊な制約がかけられている場合は、そのルールに必ず従いつつ、内部のナレッジデータ構造のみを最適化してください。
